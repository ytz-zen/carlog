# 多车辆切换方案

## 当前状态

- **DashboardPage** 已有 `cars` 列表和 `selectedCarId` 状态，也有车辆选择下拉框（仅在 `cars.length > 1` 时显示）
- **DashboardPage** 已将 `selectedCarId` 传给 Summary/Charts 等组件
- **TripsPage** (`/app/src/app/trips/page.tsx`) 完全没有车辆过滤，显示所有车辆混合行程
- **TripList** 和 **FuelLog** 组件也没有 `carId` 支持
- **API** `/api/dashboard/trips` 已支持 `carId` 参数
- **API** `/api/fuel` 不支持 `carId`

## 改动方案

### 1. API 层

| 文件 | 改动 |
|------|------|
| `app/src/app/api/dashboard/trips/route.ts` | 已支持 `carId`，无需改动 |
| `app/src/app/api/fuel/route.ts` | 加 `carId` 参数过滤（通过 tank → car 关联） |

### 2. 组件层

| 文件 | 改动 |
|------|------|
| `app/src/app/trips/page.tsx` | 加 `cars` 加载 + 顶部车辆选择下拉 + API 传 `carId` |
| `app/src/components/TripList.tsx` | 加 `carId?` prop，传参到 API |
| `app/src/components/FuelLog.tsx` | 加 `carId?` prop，传参到 API |
| `app/src/components/DashboardPage.tsx` | 车辆下拉改为始终显示（1 辆车也显示）；`<TripList>` 和 `<FuelLog>` 传 `carId` |

### 3. 实施顺序

1. 改 API: `/api/fuel?carId=xxx` 支持
2. 改 TripList: 加 `carId` prop
3. 改 FuelLog: 加 `carId` prop
4. 改 DashboardPage: 传 carId 给 TripList/FuelLog，下拉框 1 辆车也显示
5. 改 TripsPage: 完整的车辆选择 + 过滤

### 4. TripsPage 设计

```tsx
// 顶部区域
<header>
  <button onClick={() => router.push('/')}>←</button>
  <h1>🚗 行程记录</h1>
  {cars.length > 1 && (
    <select value={selectedCarId || ''} onChange={e => setSelectedCarId(e.target.value || null)}>
      <option value="">全部车辆</option>
      {cars.map(c => <option key={c.id} value={c.id}>{c.name}</option>)}
    </select>
  )}
</header>

// 表格增加"车辆"列（多车时显示）
<th>车辆</th>
```
